NAME: Neccesary Optional Attribute.

CLASSIFICATION: Good Practice.

MOTIVATION: Optional attributes in certatin configurations are very important in every domain. For example, we can have new gene product that we know that performs one of two functions (Z and Y), but only when it performs function X.

AIM: To model the fact that an entity can have different configurations of attributes, for example if having attribute A and another attribute, it must be attribute B.

STRUCTURE:

SAMPLE:

ELEMENTS: The main element is the parent class that sets the conditions of optionality: some attributes will be neccesary, some optional, and the subclasses will have to commit to such constrains.

IMPLEMENTATION: All the conditions must be set in the parent class. For each of the subclasses, add the pertinent attributes, performing reasoning at each step: not committing classes should be flagged as inconsistent.

RESULT: The optional attributes are correctly represented, and any entity that does not commit to the scheme is flagged as inconsistent.
