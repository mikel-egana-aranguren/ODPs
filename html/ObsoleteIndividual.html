NAME: Obsolete Individual.

CLASSIFICATION: Extension.

MOTIVATION: In many bio-ontologies entities are obsolete but are maintained for backwards compatibility with annotators. That is usually done by codifying the obsolete value (true or false) in an annotation value. This ODP uses instead Object Properties and chains to capture the fact that, for example, an individual is obsolete, the individuals which it is related to most likely will be obsolete as well, or at least they should be revised.

AIM: To model obsolete entities.

STRUCTURE:

SAMPLE:

ELEMENTS: The most important elements are the object properties obsolete and probably_obsolete, the latter being a superproperty of the former.

IMPLEMENTATION: Create the object property probably_obsolete, and then as a subproperty obsolete. For each object property p, build a chain like inv(p) o obsolete -> probably_obsolete. Add obsolete value true or false appropiately to each individual.

RESULT: The obsolete state of each individual (true or false) is captured and it propagates along the desired object properties.

ADDITIONAL INFORMATION: This ODP can only be used with individuals, which is really limiting.
